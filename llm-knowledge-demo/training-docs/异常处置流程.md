# 异常处置流程

当行李输送设备报警时，值守人员应先查看监控画面和报警代码，判断是堵包、急停、通信中断、电机过载还是传感器异常。不同类型异常应按对应流程处理。

堵包处理流程：确认报警位置，通知现场人员到达后暂停相关输送段，移除卡阻行李，检查输送带和滚筒状态，确认无人员停留后复位设备。复位后观察至少三分钟，确保行李流恢复稳定。

急停触发流程：必须先确认触发原因。只有在现场安全风险解除、相关人员撤离设备区域后，才允许复位急停。禁止在原因不明时直接复位。

通信中断流程：检查控制柜网络状态、交换机指示灯和上位机连接状态。如果局部区域离线，应通知自动化或弱电维护人员协同排查。

所有影响航班保障的异常，需要同步通知运行控制席位，并在事件记录中标注影响范围和恢复时间。
